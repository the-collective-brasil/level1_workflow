Phase 3 — Make.com processing
Make.com receives media
Calls doGet?phone=+5511... · Apps Script scans all group_profile tabs · matches B10 · returns teacher_id + group_id
Phone lookup Teacher is identified by their WhatsApp number, not a login. Apps Script scans every group_profile_* tab looking for a match in cell B10.
Teacher confirms child + lesson stage
2 button taps · which child? · which lesson stage?
Teacher-facing Two interactions total. Button labels are drawn from the student list in the group profile — the teacher never types a name.
Optional: voice note or short text
Anything about this moment worth remembering · teacher can skip
Teacher-facing If audio is sent, Make.com transcribes via OpenAI Whisper. Stored as teacher_notes in the learning_log row and used as context in the Level 2 DIE reflection.
Make.com downloads + renames + uploads
WhatsApp URL expires ~5 min · renamed Ang_CT_0704.mp4 · uploaded to evidence folder in Google Drive
File naming Format: student_id_lessonstagecode_ddmm.ext. Lesson stage codes (CT, FT, ST etc.) are defined per teacher in their group profile tab.
Make.com calls doPost — L1 payload
teacher_id · group_id · student_id · date · lesson_stage · files link · teacher_notes
Payload DIE columns (LO_code, describe, interpret, evaluate, assessment) are not included. Apps Script leaves them blank — they are filled later if the teacher completes a Level 2 reflection.
Apps Script runs in sequence
Apps Script — 4 sequential operations
01
Appends row to learning_log
DIE columns G, I, J, K, L left blank. One row per capture. Timestamp auto-written.
02
Writes U to activity_tracker
U = upload. Written against today's date in teacher's column. Flags if 3 or more consecutive school days show zero activity.
03
Increments observation count
Student × lesson stage matrix. Cell colour darkens with each entry — coverage gaps visible at a glance without querying.
04
Checks and awards badges
e.g. 1st Upload, Full Coverage, 5-Day Streak. Badge name + date written to group_profile tab.
Apps Script All four operations run on a single doPost call. No round-trips. Each is scoped by teacher_id — no operation touches another teacher's data.
Apps Script returns response
Status · observation count · badges awarded · top 3 coverage gaps (live scan — not stored)
Coverage gaps Calculated fresh from the learning_log on every doPost response. Never written to the sheet. Make.com reads them from the response payload only.
Make.com sends confirmation to teacher
"Tagged and added to your Learning Log." · any new badges included
Teacher-facing Confirmation is the teacher's reward signal for this capture. Badge notifications are included inline if earned. No separate message.
End of loop — two paths
L1 teacher (or teacher ignores GPT link)
Loop ends for today
Repeats from Phase 1 the following school day. No further prompts today.
L2 teacher (Level 2 Ready badge earned)
GPT link included in confirmation
Teacher opens when ready. Level 2 DIE reflection begins. Nothing is time-pressured.
L1 row complete in learning_log
Evidence captured · observation count updated · activity logged · badges checked
